## Bug #4: 关键字参数解析错误

**版本**: v1.3.0  
**状态**: ✅ 已修复 (v1.3.1)

**现象**:
```
find_files("test*.txt", directory=".")
# 错误：目录不存在 G:\...\directory="."
```

**根因**: 参数解析器不识别 `key=value` 格式，把整个 `directory="."` 当作字符串参数

**解决**:
1. `_parse_single_arg` 检测 `key=value` 格式，返回 `(key, value)` 元组
2. `execute` 方法分离位置参数和关键字参数

## Bug #3: 模型输出格式错误

**版本**: v1.3.0  
**状态**: ✅ 已修复 (v1.3.1)

**现象**:
```
💭 思考：用户想了解我有哪些可用工具...
❌ 错误：模型未输出有效的 Action
```

**根因**: 模型输出 `<thought>` 后停止，没有输出 `<action>` 或 `<final_answer>` 标签。

**解决**:
1. 加强 Prompt 格式要求，明确说明必须同时输出
2. 改进错误提示，显示模型原始输出

## Bug #2: 权限检查时转义警告

**版本**: v1.1.0 - v1.1.4  
**状态**: ⏸️ 暂缓（不影响功能）

**现象**: 
```
⚠️ 权限检查错误: incomplete escape \U at position 2
```
但操作正常执行成功。

**根因**: 未完全定位，疑似某处字符串处理触发了 Python 转义解析。

**临时方案**: try/except 已捕获错误，不影响功能执行。

**优先级**: 低（只是警告提示，不影响使用）
